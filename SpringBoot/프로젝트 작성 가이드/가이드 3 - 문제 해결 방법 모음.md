### 인텔리제이 프로젝트 인식 안될 때

* Gradle Wrapper 항상 사용

    - IDE 설정: File > Settings > Build, Execution, Deployment > Build Tools > Gradle

    - Use Gradle from: gradle-wrapper.properties로 지정

* IDE 캐시 손상 시 수동 초기화

    - File > Invalidate Caches / Restart... → “Invalidate and Restart” 클릭

* 프로젝트 구조 확인

    - File > Project Structure > Modules 에서 src/main/java, resources가 있는지 확인

* *.idea / .iml 삭제 후 재열기 (마지막 수단)

    - 문제가 반복되면 .idea 폴더와 .iml 파일을 삭제하고, IntelliJ에서 “Open as Project”로 다시 열면 깨끗하게 재생성됨


### 프로젝트

* RestController 포인트컷 적용 시 레스트컨트롤러에서 서비스 계층의 메서드 실행 시 NPE 발생

    1. Spring AOP는 public / protected 메서드만 프록시 가능
    2. private 메서드는 프록시를 거치지 않는다
    3. 프록시를 거치지 않으면 DI가 적용되지 않은 원본 객체가 호출될 수 있다

        - 애플리케이션 실행 시 포인트컷을 적용해 프록시 생성

        - 프록시 내부에 DI 가 적용된 타깃 객체 주입

    4. 원본 Controller 인스턴스의 userService는 null
    5. 그래서 private 메서드 호출 시 NPE가 발생했다
    6. 메서드를 public 으로 바꾸자 AOP가 적용되기 시작했고, 프록시 객체를 통해 정상 DI를 갖는 타깃 객체가 호출되었다 → 해결

    - 흐름

        - 애플리케이션 실행 시 포인트컷 대상인 컨트롤러는 정상적으로 프록시 빈으로 등록됨

        - 컨트롤러의 private 메서드 요청 시 프록시 컨트롤러 객체가 호출되지만, private 메서드는 AOP 대상이 아니므로 호출하지 못하고 원본 객체 바로 호출

            - 동일한 컨트롤러의 다른 public 메서드는 요청 시 요청 -> 프록시 객체 -> 타겟 호출(원본 객체) 순으로 정상적으로 처리됨

        - private 메서드 실행 시 호출된 원본 컨트롤러 객체는 DI 가 적용되지 않았으므로 Service 필드는 Null

        - Service 객체의 메서드를 호출하는 순간 NPE 발생
