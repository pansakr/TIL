### 개발 프로세스

* 요구사항 정의

    - “무엇을 만들까?” 정리 → 기획 단계

    - 예: 사용자는 구독 상품을 등록/삭제할 수 있어야 한다, 관리자는 회원을 관리할 수 있어야 한다.

* 흐름도(Use Case / User Flow) 작성

    - 다이어그램 같은 형태

    - 사용자(User)와 관리자(Admin)를 나눠서 어떤 기능이 있는지, 어떤 순서로 동작하는지 그림으로 표현

* DB 설계 (ERD)

    - 기능 흐름을 보고 필요한 데이터 테이블 설계

    - 회원, 구독 상품, 구독 내역, 결제, 알림 등 테이블 정의

* 와이어프레임/UI 시안

    - Figma 같은 툴로 화면 설계 (로그인 화면, 내 구독 화면, 관리자 대시보드 등)

    - 백엔드 개발자는 이 단계에서 API 명세를 구체화

* API 설계 (명세서)

    - REST API 또는 GraphQL → 어떤 요청을 보내고, 어떤 응답을 받을지 정의

    - Swagger/OpenAPI 문서로 자동화하는 경우 많음

* 개발/구현

    - 백엔드(Spring Boot, DB, 배포) + 프론트(React, Vue 등) + 인프라(AWS, Docker)

* 테스트/배포/운영

    - 단위 테스트, 통합 테스트

    - AWS 등 클라우드 환경에 배포 → 모니터링/로그 수집
