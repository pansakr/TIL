### Spring Boot 동작 순서 (Java 실행 순서 비유)

* 빌드 결과물 생성

  - .java → .class 컴파일 (Gradle/Maven 빌드 과정)

  - 빌드 시 META-INF/MANIFEST.MF에 메인 클래스 정보가 포함된 실행 가능한 .jar 파일이 만들어짐

  - 이때 Spring Boot 플러그인이 spring-boot-loader 관련 클래스를 함께 포함시킴

* JVM 구동 & 클래스 로딩 시작

  - java -jar app.jar 실행 시, JVM이 JarLauncher (Spring Boot의 부트스트랩 클래스)를 **메서드 영역(Method Area)**에 로드

  - JarLauncher가 META-INF/MANIFEST.MF에 지정된 메인 클래스를 찾아 실행

  - SpringApplication.run() 호출

  - 메인 클래스(@SpringBootApplication 붙은 클래스)의 main() 메서드가 실행됨

  - SpringApplication 객체 생성 → run() 메서드 호출

* 스프링 컨테이너 준비

  - SpringApplication이 ApplicationContext(스프링 컨테이너)를 만들 준비를 함

  - 웹 애플리케이션이라면 AnnotationConfigServletWebServerApplicationContext 같은 웹 전용 컨텍스트 선택

  - Banner 출력 (터미널에 나오는 Spring 로고)

* 환경(Environment) 설정

  - OS 환경변수, application.properties/application.yml 등을 로드해 Environment 객체 구성

* 빈(Bean) 등록 시작

  - @SpringBootApplication → @ComponentScan 동작

  - @Component, @Service, @Repository, @Controller, @Configuration 등 스캔 후 Bean 정의를 컨테이너에 등록

  - AutoConfiguration 동작

  - @EnableAutoConfiguration에 의해 spring.factories 파일을 읽음

  - JDBC, JPA, WebMVC, Security 등 조건부 설정(@ConditionalOn...)을 만족하는 Bean 자동 등록

* 내장 톰캣(서버) 구동

  - 웹 애플리케이션이라면 TomcatServletWebServerFactory Bean 생성

  - 내장 톰캣 인스턴스 초기화 → 포트 바인딩 → 요청 대기 상태 진입

* 빈 초기화 & 의존성 주입(DI)

  - @Autowired, 생성자 주입 등으로 Bean 사이의 의존성 연결

  - @PostConstruct 메서드 실행

  - CommandLineRunner / ApplicationRunner 실행

  - 컨텍스트가 완전히 초기화된 후, 필요한 시작 로직 실행

* 애플리케이션 실행 대기

  - 서버가 요청을 받을 준비 완료

  - HTTP 요청이 오면 DispatcherServlet이 컨트롤러 매핑 → 서비스 호출 → 결과 반환

* 종료 시 동작

  - SIGTERM(kill), Ctrl+C 등으로 종료 시 스프링이 빈 소멸 과정 실행 (@PreDestroy, DisposableBean)

  - 내장 톰캣 종료 → JVM 종료

### Spring Boot 실행 핵심 순서

* 빌드

    - .java → .class 컴파일 후 실행 가능한 .jar(fat jar) 생성

    - spring-boot-loader와 모든 의존성 포함

* JVM 실행 & 메인 클래스 로딩

    java -jar app.jar → JVM이 JarLauncher 실행

    @SpringBootApplication main 메서드 호출 → SpringApplication.run() 시작

* 스프링 컨테이너 준비

    - 환경(Environment) 로드

    - Bean 스캔 & 등록 (@ComponentScan)

    - 자동 설정(@EnableAutoConfiguration) 적용

* 내장 서버 구동 (웹일 경우)

    - 내장 Tomcat/Jetty 실행 → 포트 바인딩

* Bean 초기화 & DI

    - 의존성 주입, 초기화 메서드 실행

* 애플리케이션 실행

    - HTTP 요청 처리 (DispatcherServlet) 또는 비웹 로직 실행

* 종료 처리

    - Bean 소멸 → 내장 서버 종료 → JVM 종료
