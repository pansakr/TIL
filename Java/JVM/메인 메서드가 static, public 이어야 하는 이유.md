### 메인 메서드가 static 이어야 하는 이유

* 메인 메서드가 static 이 아니면 JVM 이 객체를 만들어야 하는데 JVM 은 임의로 객체를 생성할 수 없으므로 프로그램을 시작할 수 없다

    1. JVM 이 프로그램 실행 전에는 객체가 없는 상태

        - main() 이 실행 되어야 프로그램이 시작되고, 객체가 생성될 수 있음
    
    2. JVM 은 프로그램 시작점으로 특정 시그니처의 메서드를 호출하도록 설계되어 있다

        ```java
        // 정확하게 아래 시그니처를 찾음
        public static void main(String[] args)

        // 찾아서 호출
        MainClass.main(args)    // static 으로 선언되어 있어 클래스.메서드() 방식으로 호출 가능
        ```

    3. main() 이 static 이 아니라면?

        - 인스턴스 메서드는 객체가 없으면 호출할 수 없기 때문에 객체를 먼저 생성해야 한다

        ```java
        // 인스턴스 메서드
        public void main(String[] args)

        // 이런 방식으로 호출해야 함
        new MainCalss().main(args);
        ```

    4. 하지만 JVM 은 임의로 객체를 만들 수 없다 (생성자 임의 호출 금지됨)

        - JVM 은 프로그래머의 명시적 요청이 없이 객체를 생성할 권한이 없다(Java 언어 스펙에서 금지)
        
            - 객체 생성은 반드시 new, 리플렉션 API 등 프로그래머가 '의도를 표현한 코드' 가 있어야 가능하다

            - JVM 이 임의로 객체를 만들 수 없는 이유

                - 생성자가 여러개일 때 어떤 생성자를 호출해야 하는가?

                - 매개변수를 무엇으로 채워야 하는가?
    
    5. 따라서 JVM 은 객체를 만들 수 없으니 mian() 을 실행하지 못하고, 프로그램을 시작할 수 없게 된다
    
- 객체가 없어도 호출 가능한 static main() 만이 프로그램 시작점으로 적합하다


### public 이어야 하는 이유

* 패키지/클래스 밖에 있는 JVM 이 main 메서드를 정상적으로 호출할 수 있도록 하기 위함

### void 인 이유

* main() 메서드가 끝나면 자바 프로그램 역시 끝나기 때문에 리턴값이 필요 없음
