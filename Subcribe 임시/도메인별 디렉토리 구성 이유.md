### 도메인 별 디렉토리 구성 이유

* 관련된 기능별로 코드가 모여 있어서 유지보수가 편함

* controller, service, repository 처럼 계층별로 디렉토리를 구성하는 방법도 있지만, 기능이 많아질수록 관련된 클래스들이 폴더 여러 곳에 흩어져서 코드를 파악하기가 점점 어려움

* 그래서 회원, 구독, 주문 같은 도메인 단위로 묶어서 관련 클래스들을 한 폴더에 두면 기능을 수정할 때 그 폴더만 보면 되니 관리가 훨씬 수월함

* 도메인 별 디렉토리 구성의 단점

    - 처음 디렉토리를 구성할 때 도메인 경계를 어떻게 나눌지 고민하는 시간이 필요함

    - 하지만 한번 구조를 잡아두면 기능이 점점 많아져도 확장을 체계적으로 할 수 있어서 결과적으로는 더 관리하기 편함

        - 도메인 경계를 정한 방법

            - ERD 기준으로 주요 테이블 단위로 기능을 구분

* 디렉토리 구성 방식 종류

    - 레이어 단위

    - 도메인 단위

    - 마이크로서비스
